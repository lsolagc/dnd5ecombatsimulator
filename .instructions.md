# Instruções para Agentes de IA - D&D Character Manager

## 📋 Resumo do Projeto

**D&D Character Manager** é um projeto Rails para gerenciar personagens e combates de Dungeons & Dragons 5e.

- **Stack**: Rails 8 + Phlex + Tailwind
- **Banco**: SQLite/PostgreSQL
- **Principais features**: Personalização de classes, criação de personagens, sistema de combate

| `PlayerCharacter` | Personagem do jogador com ability scores | [app/models/player_character.rb](app/models/player_character.rb) |

### Serviços

| Serviço | Responsabilidade | Localização |
|---------|------------------|-------------|
| `EncounterService` | Orquestra combates completos no estado atual; serviço provisório, acoplado ao restante do sistema, não alterar fora de refatoração planejada | [app/services/encounter_service.rb](app/services/encounter_service.rb) |
| `LevelUpService` | Aplica aumentos de nível | [app/services/level_up_service.rb](app/services/level_up_service.rb) |

### Restrições Importantes

- `EncounterService` deve ser tratado como implementação temporária. Se uma tarefa tocar nesse serviço, o padrão esperado é atualizar documentação ou trabalhar ao redor dele, não expandir sua lógica.
- Mudanças em `EncounterService` só devem acontecer quando o escopo incluir explicitamente a refatoração do sistema de combate.

### Lib Utilities

| Módulo | Responsabilidade | Localização |
|--------|------------------|-------------|
| `Dice` | Rolar dados genéricas | [lib/dice.rb](lib/dice.rb) |
| `Dice::AttackRoll` | Rolar ataque (d20 + mods) | [lib/dice/attack_roll.rb](lib/dice/attack_roll.rb) |

## 💡 Padrões de Desenvolvimento

### Regra Global de Migrations

- Sempre criar migrations via gerador do Rails (`bin/rails generate migration ...` ou `rails g migration ...`).
- Depois de gerar, editar o arquivo em `db/migrate/` com as mudanças necessárias.
- Não criar migrations manualmente com timestamp e nome de arquivo escritos à mão.

### 1. Adicionar Feature em Model

```ruby
# app/models/player_character.rb
class PlayerCharacter < ApplicationRecord
  belongs_to :player_class
  validates :name, presence: true
  validates :ability_scores, inclusion: { in: 1..20 }
  
  def strength_modifier
    (strength - 10) / 2
  end
end
```

### 2. Lógica Complexa → Service

```ruby
# app/services/my_service.rb
class MyService
  def initialize(param)
    @param = param
  end
  
  def call
    # lógica aqui
    result
  end
end

# Uso: MyService.new(param).call
```

### 3. View em Phlex Component

```ruby
# app/components/my_component.rb
module Components
  class MyComponent < Base
    def initialize(data)
      @data = data
    end
    
    def template
      div(class: "component") { render_content }
    end
    
    private
    
    def render_content
      # HTML aqui
    end
  end
end
```

### 4. Controller - Ação Padrão

```ruby
# app/controllers/my_resources_controller.rb
class MyResourcesController < ApplicationController
  def index
    @items = MyResource.all
  end
  
  def show
    @item = MyResource.find(params[:id])
  end
  
  def create
    @item = MyResource.new(item_params)
    if @item.save
      redirect_to @item, notice: 'Created'
    else
      render :new
    end
  end
  
  private
  
  def item_params
    params.require(:my_resource).permit(:name, :description)
  end
end
```

## 🚀 Tarefas Comuns

### Adicionar Campo a Model

1. Criar migration:
   ```bash
   rails g migration AddFieldToModel field_name:type
   ```

2. Editar migration em `db/migrate/`

3. Rodar:
   ```bash
   rails db:migrate
   ```

4. Atualizar model validações

## 🔍 Como Explorar o Código

### Entender um Fluxo

```
1. Localize o controller: app/controllers/*/
2. Veja a ação: def create, def show, etc
3. Encontre o model: app/models/
4. Procure a view/component: app/components/ ou app/views/
5. Se há lógica complexa: app/services/
```

### Exemplo: Criando Personagem

```
GET /player_characters/new
  ↓
PlayerCharactersController#new
  ↓
render :new (busca view)
  ↓
app/views/player_characters/new.html.erb
  ↓
Renderiza form do model

POST /player_characters
  ↓
PlayerCharactersController#create
  ↓
@character = PlayerCharacter.new(character_params)
@character.save
  ↓
app/models/player_character.rb (validações, callbacks)
  ↓
Redirect v/error
```

### Buscar Onde X é Usado

```bash

# Exemplo: encontrar uso de Combatant
grep -r "Combatant" app/
```

## ⚠️ Cuidados

1. **Migrations é history**: Nunca edite migrations antigas, crie novas
2. **Validações no model**: Sempre valide em modelo, não só controller
3. **Polimorfismo em Combatant**: Se adicionar tipo, testar com ambos tipos
4. **Hit die congelado**: ClassLevelProgression é freeze, dados imutáveis por design
5. **Testes primeiro**: Escreva testes antes de features complexas
